## 2. CUDA 13.0.2 설치

PyTorch 2.x + Flash Attention 2가 안정적으로 동작하는 최신 CUDA 이어야 합니다. NVIDIA 공식 아카이브에서 13.0.2를 받아 설치합니다.

다운로드 페이지: <https://developer.nvidia.com/cuda-13-0-2-download-archive>

### 2.1 Windows 설치

1. 위 페이지에서 **Operating System: Windows → Architecture: x86_64 → Version: 11/10 → Installer Type: exe (local)** 선택.
2. 받은 `cuda_13.0.2_windows.exe` 실행 → "Express (권장)" 설치.
3. 설치 후 PowerShell에서 확인:

   ```powershell
   nvidia-smi
   nvcc --version
   ```

4. `nvcc`가 인식되지 않으면 시스템 환경 변수 `Path`에 다음을 추가:

   ```
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.0\bin
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.0\libnvvp
   ```

5. 재부팅한 뒤 다시 `nvcc --version`으로 확인.

> **드라이버 주의**: CUDA 13.x는 NVIDIA 드라이버 580 이상이 필요합니다. 구형 드라이버라면 GeForce Experience 또는 NVIDIA 다운로드 센터에서 먼저 업데이트하세요.


## 5. 데이터셋 준비

Anima LoRA는 *이미지 + 같은 이름의 `.txt` 캡션 사이드카* 구조를 사용합니다. 아래는 `image_dataset/` 폴더의 예시입니다.

```
image_dataset/
├─ 00001.png
├─ 00001.txt
├─ 00002.jpg
├─ 00002.txt
├─ subfolder/
│  ├─ 00010.webp
│  └─ 00010.txt
└─ ...
```

### 5.1 캡션 작성 팁

- anima 공식 가이드라인을 따르면 태그 순서는 항상 [meta] [character] [series] [artist] [general] 입니다. 예를 들어

```
absurdres, safe, 1girl, chitanda eru, hyouka, @channel (caststation), full body, serafuku, She is saying hi.
```
와 같이 작성하시면 됩니다.

- 이 때 absurdres, highres, masterpiece 등의 퀄리티 태그는 개인적으로 실험한 결과 작성하지 않거나 최소한으로만 남기는 것이 제일 좋습니다. 혹은 추후 정식 배포될 mod guidance를 사용하면 아예 작성하지 않아도 됩니다.
- 원본 이미지는 `image_dataset/`에 둡니다 (이름은 자유, 이 위치를 그대로 사용하세요).

### 5.2 `num_repeats`는 무엇이고 언제 만지나요? (요약: **만지지 않습니다**)

`configs/base.toml`의 `[[datasets.subsets]]` 안에 `num_repeats = 1` 항목이 보일 겁니다. 이 값은 **한 에폭 동안 각 이미지를 몇 번 반복해서 사용할지**를 지정하는 kohya-ss 스타일 옵션으로, 다른 LoRA 트레이너 가이드를 따라하다 보면 자주 마주치는 키입니다.

- **이 가이드북의 표준 워크플로우에서는 그냥 `1`로 두세요.** 모든 이미지를 `image_dataset/` 한 폴더에 모아 학습하는 일반적인 사용 패턴에서는 `num_repeats`를 올려도 *한 에폭이 길어질 뿐* 효과는 `max_train_epochs`를 늘리는 것과 같습니다. 학습량은 항상 에폭 수로 조절하는 것이 직관적이며, 본 프로젝트의 모든 프리셋·메소드 설정은 `num_repeats = 1`을 전제로 튜닝되어 있습니다.
- **언제 올리는 게 의미가 있나요?** 한 학습에 *서로 다른 서브셋(폴더)이 여러 개* 있고 폴더별 이미지 수가 크게 차이날 때, 작은 폴더의 노출 빈도를 큰 폴더에 맞추기 위한 *밸런싱 도구*로만 의미가 있습니다 (예: 캐릭터 A 1000장 + 캐릭터 B 50장이라면 B 서브셋만 `num_repeats = 20`). 단일 폴더 학습에는 해당되지 않습니다.
- **어디서 바꾸나요?** `num_repeats`는 메소드 설정이 아니라 *데이터셋 설정*이라 `configs/methods/`·`configs/gui-methods/`·GUI 학습 탭에는 노출되지 않습니다. 정말 필요하다면 `configs/base.toml`의 `[[datasets.subsets]]`(또는 `--dataset_config <path>`로 따로 지정한 TOML)를 직접 편집하세요. *동일 이미지를 단순히 더 많이 학습시키고 싶다면* `num_repeats`가 아니라 `max_train_epochs`를 늘리는 것이 정답입니다.

## 6. 전처리: 리사이즈 · 잠재변수 · 텍스트 임베딩 캐싱

학습 속도와 VRAM을 최적화하기 위해 **리사이즈 → VAE 잠재 캐싱 → 텍스트 임베딩 캐싱** 세 단계를 사전 수행합니다.

```bash
make preprocess              # 세 단계 모두 실행 (LoRA / 일반 학습용)
# 또는 단계별로
make preprocess-resize       # 1) image_dataset/ → post_image_dataset/resized/
make preprocess-vae          # 2) VAE 잠재변수 캐싱 → post_image_dataset/lora/
make preprocess-te           # 3) 텍스트 인코더 출력 캐싱 → post_image_dataset/lora/
make preprocess-pe           # (선택) PE-Core 비전 인코더 피처 캐싱 — IP-Adapter / REPA 전용
```

### 6.1 리사이즈가 하는 일

- VAE가 요구하는 픽셀 정렬에 맞춰 이미지를 리사이즈
- (H/16) × (W/16) ≈ 4096 패치가 되는 *고정 토큰 버킷*에 자동 분류
- 너무 작은 이미지(기본 0.5MP 미만)는 자동 제외하고 보고
- 결과는 `post_image_dataset/resized/`에 PNG로 저장

### 6.2 잠재변수 캐싱

- 모든 리사이즈 이미지에 대해 VAE를 한 번만 돌려 결과를 디스크에 저장
- 학습 중에는 VAE를 GPU에 올리지 않으므로 VRAM이 크게 절약됨
- 캐시 위치: `post_image_dataset/lora/{stem}_{WxH}_anima.npz`
- 스크립트: `preprocess/cache_latents.py`

### 6.3 텍스트 임베딩 캐싱

- Qwen3 0.6B + LLM 어댑터 출력을 미리 계산
- `use_shuffled_caption_variants = true`이면 캡션 콤마 셔플 변형까지 같이 캐싱 (학습 시 무작위 선택)
- 캐시 위치: `post_image_dataset/lora/{stem}_anima_te.safetensors`
- 캡션은 항상 `image_dataset/`의 원본 `.txt`를 읽습니다 (리사이즈 폴더에 복사하지 않음)
- 스크립트: `preprocess/cache_text_embeddings.py`

### 6.4 PE 비전 피처 캐싱 (선택)

- IP-Adapter와 REPA(`use_repa = true`)에 필요
- PE-Core-L14-336 비전 인코더 출력을 미리 계산해 학습 중 비전 인코더를 올리지 않도록 함
- 캐시 위치: `post_image_dataset/lora/{stem}_anima_pe.safetensors`
- IP-Adapter를 쓸 거면 `make ip-adapter-preprocess` 한 번이 `make preprocess` + `make preprocess-pe`를 모두 돌려줍니다.

### 6.5 EasyControl용 별도 데이터셋

EasyControl만 다른 소스 폴더를 씁니다:

```bash
make easycontrol-preprocess  # easycontrol-dataset/ → post_image_dataset/easycontrol/
```

> **캐시는 언제 다시 만들어야 하나요?**
> - 캡션을 수정했거나 이미지가 추가/삭제되었을 때
> - 토크나이저/패딩 관련 옵션을 바꿨을 때
> `post_image_dataset/lora/`(또는 `post_image_dataset/easycontrol/`)를 통째로 지우고 다시 돌리면 됩니다.

## 7. GUI 사용법

PySide6 기반 GUI를 사용하면 설정 편집, 데이터셋 탐색, 학습 실행/모니터링, IP-Adapter / EasyControl 전처리·학습, LoRA 병합을 한 화면에서 할 수 있습니다.

```bash
make gui
make gui-shortcut   # (선택) 윈도우 바탕화면에 콘솔 창 없는 .lnk 바로가기 생성
```

GUI 주요 탭:

- **학습 설정 (Training Config)**: LoRA 계열 변형(`lora`, `ortholora`, `tlora`, `hydralora`, `reft`)을 드롭다운으로 선택하고, `presets.toml` 프리셋(default / low_vram 등)과 모든 학습 키를 직접 수정해 그대로 학습 시작
- **메소드 (Methods)**: 실험적 변형(`postfix`, `apex`)을 같은 방식으로 편집 · 실행
- **IP-Adapter**: 전처리(리사이즈 + VAE + TE + PE 캐시)와 학습을 한 탭에서
- **EasyControl**: `easycontrol-dataset/` 전처리와 학습을 한 탭에서
- **데이터셋 (Dataset)**: 이미지/캡션을 미리보고 캡션을 직접 수정
- **병합 (Merge)**: 학습된 LoRA를 베이스 DiT에 베이크해 ComfyUI 단독 체크포인트로 저장 (기본 LoRA / OrthoLoRA / T-LoRA만 지원)

GUI 학습도 결국 내부적으로 `train.py`를 호출하므로, 같은 파라미터를 CLI에서도 그대로 재현할 수 있습니다. GUI는 `configs/gui-methods/<variant>.toml`(토글 블록 없는 변형별 단일 파일)을 읽으므로, GUI에서 노출되는 변형 목록과 CLI의 `make lora-gui GUI_PRESETS=<variant>`가 같은 결과를 냅니다.

### 7.1 폼 편집과 저장 동작

학습/전처리 서브프로세스는 디스크의 variant TOML 파일을 다시 읽어들이기 때문에, 폼만 수정하고 저장하지 않으면 그 변경 사항은 학습에 반영되지 않습니다. GUI는 이를 두 가지로 처리합니다.

- **변경 감지**: 폼의 어떤 필드라도 (또는 `+ Extra args` 텍스트박스를) 편집하면 `Save` 버튼이 주황색으로 바뀌고 `Save *` 표시가 붙습니다. 즉 *디스크의 variant 파일과 화면이 다르다*는 신호입니다. `Save`를 누르거나 variant를 다시 선택해서 디스크에서 다시 불러오면 표시가 사라집니다.
- **자동 저장**: 그래도 저장을 깜빡하고 `Train` / `Preprocess`를 누르면, 해당 시점의 폼 값이 자동으로 variant 파일에 먼저 기록된 뒤 서브프로세스가 실행됩니다. 즉 화면에 보이는 값이 곧 학습에 쓰이는 값입니다. (`Test`는 마지막 학습 결과 체크포인트로 추론하므로 자동 저장 대상이 아닙니다.)

> 변경 사항을 *시험만* 해보고 싶다면, 폼을 편집한 뒤 `Train`을 누르지 말고 variant를 한 번 다른 것으로 바꿨다 돌아오세요 — 디스크에서 다시 로드되어 편집이 폐기됩니다.

### 7.2 자동 재개 (checkpointing_epochs)

학습이 도중에 끊겨도 다시 `Train`을 누르면 **자동으로 마지막 저장 지점부터 이어 학습**합니다. 정전·OOM·실수로 창 닫기 등에 대비할 수 있는 가장 유용한 기능 중 하나이므로 기본값으로 켜져 있습니다.

GUI에서 다루는 방법:

- 학습 설정 탭의 **Training 그룹**에 `checkpointing_epochs` 필드가 있습니다 (기본 LoRA에서 `2`). N 에폭마다 재개용 상태를 저장하며, 같은 파일을 덮어쓰므로 디스크가 부풀지 않습니다.
- 학습이 끊긴 뒤 같은 variant로 다시 `Train`을 누르면, 로그 창에 `auto-resuming from checkpoint at step N`이 뜨고 그 지점부터 이어 학습합니다. 별도 플래그를 손댈 필요 없습니다.
- 학습이 정상적으로 끝나면 재개용 임시 파일은 자동 삭제되고, 최종 결과는 `output/ckpt/<output_name>.safetensors`로 남습니다.
- **데이터셋이나 핵심 설정(랭크·LR·에폭 수 등)을 바꾼 뒤** 새로 학습하려면 `output/ckpt/<output_name>-checkpoint-state/` 폴더를 수동으로 지운 다음 `Train`을 누르세요. 그렇지 않으면 옛 상태 위에 학습이 이어집니다.

상세 동작은 [§8.6 자동 재개](#86-자동-재개-checkpointing_epochs)에서 다룹니다 — `save_every_n_epochs`와의 차이점도 거기서 정리.

## 8. 학습 실행

모든 학습은 TOML 설정 파일과 HuggingFace Accelerate를 통해 실행됩니다. 설정 병합 순서는 `configs/base.toml → configs/presets.toml[<preset>] → configs/methods/<method>.toml → CLI args`이며, 메소드 설정이 프리셋을 덮어씁니다.

### 8.1 빠른 시작

```bash
# LoRA 계열 (methods/lora.toml의 토글 블록으로 변형 선택)
make lora                          # presets.toml[default]
PRESET=low_vram make lora          # presets.toml[low_vram] — VRAM 8~12GB 환경
PRESET=half make lora              # 데이터셋의 절반만 사용해 빠르게 실험

# GUI 친화적 단일 변형 파일로 직접 학습 (configs/gui-methods/<variant>.toml)
make lora-gui GUI_PRESETS=tlora              # T-LoRA만 켠 단일 설정으로 학습
make lora-gui GUI_PRESETS=hydralora_sigma    # σ-라우팅 HydraLoRA
PRESET=low_vram make lora-gui GUI_PRESETS=ortholora

# 기타 패밀리
make postfix          # postfix / postfix_exp / postfix_func / postfix_sigma / prefix
make apex             # APEX 자기적대 1-NFE distillation (사전 LoRA 가중치 필요)
make ip-adapter       # IP-Adapter (이미지 cross-attention)
make easycontrol      # EasyControl (확장 self-attn 이미지 conditioning)
```

> **CLI에서 직접 키 오버라이드**: `make lora -- --network_dim 32 --max_train_epochs 24` 처럼 추가 인자를 그대로 전달할 수 있습니다 (`tasks.py`도 동일).

### 8.2 마스크드 로스 (텍스트 버블 제외)

만화/만화풍 데이터에서 *말풍선이나 텍스트 영역*을 학습 손실에서 제외하면 결과가 훨씬 깨끗해집니다.

```bash
make mask          # SAM3 + MIT 둘 다 생성 후 병합 → masks/merged/
make mask-sam      # SAM3만 → masks/sam/
make mask-mit      # MIT/ComicTextDetector만 → masks/mit/
make mask-clean    # 모든 마스크 폴더 삭제
```

결과 PNG는 흑백입니다: **흰색(255) = 학습 대상**, **검정(0) = 제외**. 데이터셋 서브셋은 `masks/merged/`가 있으면 그것을, 없으면 `masks/sam/` → `masks/mit/` 순으로 자동으로 골라 씁니다. 마스크 파일이 없으면 그냥 무시되므로 안 만들어도 됩니다.

### 8.3 자주 만지는 설정값 (LoRA 기준 기본값)

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `network_dim` | `48` | LoRA 랭크. 클수록 표현력↑, 파라미터↑ |
| `network_alpha` | `48` | LoRA 스케일 (보통 `network_dim`과 같은 값) |
| `learning_rate` | `2e-5` | 학습률. T-LoRA/Hydra는 더 낮춰도 됨 |
| `max_train_epochs` | `12` | 데이터셋이 작을수록 에폭↑ |
| `caption_dropout_rate` | `0.1` | 캡션 일부를 빈 문자열로 대체 (CFG에 도움) |
| `use_shuffled_caption_variants` | `true` | 캡션 콤마 셔플 변형 사용 |
| `compile_mode` | `"full"` | `torch.compile` 모드 (속도↑, 첫 스텝 느림) |

각 변형(`use_ortho`, `use_timestep_mask`, `use_hydra`, `add_reft`, `use_repa` 등)은 `configs/methods/lora.toml`의 토글 블록을 주석 해제해 활성화하거나, `configs/gui-methods/<variant>.toml`에서 변형별 단일 파일을 사용하세요.

### 8.4 학습 진행 중 일어나는 일

1. 텍스트 인코더 로드 → 캐시 생성/검증 → 언로드
2. VAE 로드 → 캐시 생성/검증 → 언로드
3. DiT를 *지연 로딩*하여 캐싱 단계의 VRAM 충돌 방지
4. 어댑터 네트워크를 DiT의 attention / FFN 모듈에 패치 (변형마다 대상이 다름)
5. 노이즈 샘플링 → DiT forward → flow-matching 손실 → backward → optimizer step
6. (선택) `validation_split`로 검증 손실 측정 및 샘플 이미지 생성

### 8.5 출력물

- 학습된 가중치: `output/ckpt/<output_name>.safetensors` (변형마다 `anima_lora`, `anima_tlora`, `anima_hydra_sigma`, `anima_postfix`, `anima_ip_adapter`, `anima_easycontrol` 등으로 자동 분기)
- 체크포인트: `save_every_n_epochs` 주기마다 `output/ckpt/`에 저장 (`.snapshot.toml` 사이드카 + Hydra의 경우 `_moe` 동반 파일)
- 검증 샘플: `output/ckpt/sample/`
- 추론 결과 이미지: `output/tests/`

### 8.6 자동 재개 (checkpointing_epochs)

학습이 도중에 끊기더라도 **마지막으로 저장된 지점부터 자동으로 이어서** 재개할 수 있는 기능입니다. 정전, OOM, 실수로 창 닫기, 잠깐 PC 끄고 자고 싶을 때 등에 매우 유용합니다. 기본 메소드 파일에 이미 켜져 있으므로 따로 설정할 필요는 없습니다.

```toml
checkpointing_epochs = 2     # 2 에폭마다 재개용 상태 저장 (덮어쓰기)
```

`save_every_n_epochs`와는 역할이 다릅니다.

| 키 | 저장 대상 | 누적? | 용도 |
|---|---|---|---|
| `save_every_n_epochs` | 어댑터 가중치 (`anima_lora-000004.safetensors` 등) | **누적** (또는 `save_last_n_epochs`로 제한) | 중간 결과로 추론을 돌려 보거나 과적합 시점을 비교 |
| `checkpointing_epochs` | 학습 재개용 풀 상태 (옵티마이저 / 스케줄러 / RNG / 어댑터 가중치) | **단일 파일을 덮어씀** | 학습 중단 시 자동 이어 학습 |

동작 방식:

- **자동 저장**: `checkpointing_epochs` 주기로 `output/ckpt/<output_name>-checkpoint-state/`(상태 디렉터리) + `<output_name>-checkpoint.safetensors`(가중치)가 함께 저장되며, 이전 것을 덮어씁니다. 디스크가 부풀지 않습니다.
- **자동 재개**: 같은 명령(`make lora` 등)을 다시 실행하면, 저장된 체크포인트가 있고 아직 `max_train_steps`에 도달하지 못했다면 **알아서 이어 학습합니다**. `--resume` 같은 플래그를 직접 줄 필요 없음. 로그에 `auto-resuming from checkpoint at step N`이 보이면 정상 재개된 것.
- **자동 정리**: 학습이 정상적으로 끝나면 위 두 파일이 자동 삭제됩니다 — 재개용 상태는 임시 파일이며, 최종 결과물은 `output/ckpt/<output_name>.safetensors`(8.5 참고)입니다.
- **수동 재개**: 다른 시점의 상태로 돌아가고 싶다면 `--resume <state_dir>`를 직접 지정해도 됩니다.

> **언제 비활성화?** 짧은 실험 학습이나 디스크가 부족한 경우 `checkpointing_epochs`를 주석 처리하면 됩니다. 데이터셋이 큰 본 학습에서는 거의 항상 켜두는 것을 권장합니다.
>
> **주의**: 데이터셋, 캡션, 학습 설정(랭크·LR·에폭 수 등)을 바꿨다면 이전 체크포인트로 재개하는 것이 의미가 없거나 위험합니다. 이 경우 `output/ckpt/<output_name>-checkpoint-state/` 폴더를 수동으로 지우고 새로 시작하세요.

## 9. LoRA / 어댑터 변형 선택 가이드

| 변형 | 실행 방법 | 언제 쓰나요? |
|---|---|---|
| **LoRA** | `make lora` 또는 `make lora-gui GUI_PRESETS=lora` | 가장 안정적, 처음 시도하는 사람 |
| **LoRA (8GB)** | `make lora-gui GUI_PRESETS=lora-8gb` 또는 `PRESET=low_vram make lora` | VRAM 8~12GB |
| **OrthoLoRA** | `methods/lora.toml`에서 `use_ortho = true` 또는 `make lora-gui GUI_PRESETS=ortholora` | SVD 기반, 직교성 정규화로 안정적 학습 |
| **T-LoRA** | `methods/lora.toml`에서 `use_timestep_mask = true` 또는 `make lora-gui GUI_PRESETS=tlora` | 타임스텝별 랭크 마스킹, 디테일/스타일 균형 |
| **T-LoRA + OrthoLoRA + ReFT** | `make lora-gui GUI_PRESETS=tlora_ortho_reft` | 기본 LoRA에 위 셋을 모두 스택 — `methods/lora.toml`의 기본 조합 |
| **HydraLoRA** | `make lora-gui GUI_PRESETS=hydralora_experimental` 또는 `_sigma` | MoE 다중 헤드 라우팅, 다양한 콘셉트를 한 어댑터에 |
| **ReFT** | `methods/lora.toml`에서 `add_reft = true` 또는 `make lora-gui GUI_PRESETS=reft` | 표현 편집(Representation Fine-Tuning), 매우 적은 파라미터 |
| **REPA 보조 손실** | `methods/lora.toml`에서 `use_repa = true` 또는 `make lora-gui GUI_PRESETS=lora_repa` | PE-Core 피처와의 정렬 손실로 수렴 가속 (`make preprocess-pe` 필수) |
| **Prefix Tuning** | `make postfix`(파일 안에서 prefix 블록) 또는 `make lora-gui GUI_PRESETS=prefix` | 시퀀스 앞쪽에 학습 가능한 벡터 추가 — LoRA 대안 |
| **Postfix Tuning** | `make postfix` 또는 `make lora-gui GUI_PRESETS=postfix[_exp\|_func\|_sigma]` | cross-attention 끝에 학습 가능한 N개 벡터 (캡션 조건 / σ 조건 변형 포함) |
| **APEX (1-NFE distillation)** | `make apex` | 1~4 step 추론용 distillation. **사전 LoRA 가중치 필수** (`network_weights`) |
| **IP-Adapter** | `make ip-adapter` | 참조 이미지로 conditioning (이미지 cross-attention) |
| **EasyControl** | `make easycontrol` | 참조 이미지로 conditioning (확장 self-attention, 별도 데이터셋 사용) |

각 변형의 자세한 옵션은 [`docs/guidelines/training.md`](training.md) 및 `docs/methods/`의 개별 문서를 참고하세요.

> **호환성 메모**
> - HydraLoRA / ReFT 등 어댑터 변형은 `cache_llm_adapter_outputs = true`가 기본으로 켜져 있어야 정상 동작합니다.
> - APEX는 한 step에 DiT를 3번 forward하므로 `blocks_to_swap = 0`이 강제됩니다 (메소드 파일이 자동으로 덮어씀).
> - IP-Adapter / EasyControl은 DiT를 동결하고 어댑터 부분만 학습합니다.

## 10. 추론

### 10.1 가장 빠른 테스트

방금 학습한 어댑터로 즉시 샘플을 만들고 싶다면, 변형에 맞는 `make test-*`를 쓰세요. 모두 `output/ckpt/`에서 가장 최근에 저장된 해당 어댑터를 자동으로 골라 추론합니다.

```bash
make test                        # 일반 LoRA / OrthoLoRA / T-LoRA / ReFT
make test-spectrum               # Spectrum 가속 추론
make test-mod                    # Modulation guidance (pooled_text_proj)
make test-apex                   # APEX 4-step euler 추론
make test-hydra                  # HydraLoRA (router-live, anima_hydra*_moe.safetensors)
make test-prefix                 # Prefix tuning
make test-postfix                # Postfix tuning
make test-postfix-exp            # postfix_exp
make test-postfix-func           # postfix_func
make test-ip REF_IMAGE=ref.png   # IP-Adapter (이미지 conditioning)
make test-easycontrol REF_IMAGE=ref.png   # EasyControl (이미지 conditioning)
make test-merge                  # 베이크된 단독 DiT(`*_merged.safetensors`)로 추론
```

### 10.2 일반 추론 (수동)

```bash
python inference.py \
    --dit models/diffusion_models/anima-preview3-base.safetensors \
    --text_encoder models/text_encoders/qwen_3_06b_base.safetensors \
    --vae models/vae/qwen_image_vae.safetensors \
    --lora_weight output/ckpt/anima_lora.safetensors \
    --lora_multiplier 1.0 \
    --prompt "masterpiece, best quality, an anime girl in a sunlit forest" \
    --negative_prompt "worst quality, low quality, blurry" \
    --image_size 1024 1024 \
    --infer_steps 30 \
    --guidance_scale 4.0 \
    --sampler er_sde \
    --flow_shift 1.0 \
    --seed 42 \
    --save_path output/tests
```

자주 쓰는 플래그:

| 플래그 | 설명 |
|---|---|
| `--lora_weight` | 학습된 어댑터 경로. 여러 개 동시에 줄 수도 있음 |
| `--lora_multiplier` | 어댑터 강도 (0.0~1.5) |
| `--image_size H W` | 출력 해상도 (예: `1024 1024`, `1024 1536`) |
| `--infer_steps` | 디노이징 스텝 수 (보통 20~50) |
| `--guidance_scale` | CFG 강도 (3.0~5.0 권장) |
| `--sampler` | `er_sde`, `euler`, `dpm++` 등 |
| `--seed` | 재현용 시드 |
| `--spectrum` | Spectrum 가속 활성화 |
| `--pgraft` | P-GRAFT (디노이즈 후반 LoRA 컷오프)로 베이스 모델이 후반 디테일 담당 |

추론 옵션의 전체 목록과 P-GRAFT 추론 방법은 [`docs/guidelines/inference.md`](inference.md)에 있습니다.

## 11. ComfyUI로 배포

ComfyUI 코어는 Anima 베이스 DiT를 기본 지원합니다 (`UNETLoader` / `CLIPLoader`로 그대로 로드). 어댑터 종류에 따라 배포 방식이 다릅니다.

### 11.1 클래식 LoRA / OrthoLoRA / T-LoRA

`output/ckpt/`에서 나온 `.safetensors`를 그대로 `ComfyUI/models/loras/`에 복사하면, ComfyUI 기본 LoraLoader 노드로 바로 사용할 수 있습니다. 더 깔끔하게 단독 체크포인트로 만들고 싶다면:

```bash
make merge ADAPTER_DIR=output/ckpt                 # 가장 최근 가중치를 베이스 DiT에 베이크
make merge ADAPTER_DIR=output/ckpt MULTIPLIER=0.8  # 강도 조절
```

베이크된 `*_merged.safetensors`는 그대로 ComfyUI의 `UNETLoader`에서 단독 모델로 로드 가능합니다.

### 11.2 HydraLoRA / ReFT / Postfix / Prefix

이 변형들은 ComfyUI 기본 LoraLoader로 로드할 수 없고 (단순 weight delta가 아니라 라우팅·토큰 삽입이 들어감), 전용 노드가 필요합니다:

- **Anima Adapter Loader** (`custom_nodes/comfyui-hydralora/`) — LoRA / Hydra / ReFT / prefix / postfix를 통합 처리. 자세한 사용법은 해당 폴더의 `README.md` 참고.
- **Spectrum KSampler / Mod Guidance 노드** — 별도 저장소 <https://github.com/sorryhyun/ComfyUI-Spectrum-KSampler>

### 11.3 IP-Adapter / EasyControl

현재 ComfyUI에서의 직접 사용은 미지원입니다. `inference.py` 또는 GUI의 IP-Adapter / EasyControl 탭을 사용하세요.
