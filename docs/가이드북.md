|---|---|---|

## 2. CUDA 13.0.2 설치

PyTorch 2.x + Flash Attention 2가 안정적으로 동작하는 최신 CUDA 이어야 합니다. NVIDIA 공식 아카이브에서 13.0.2를 받아 설치합니다.

다운로드 페이지: <https://developer.nvidia.com/cuda-13-0-2-download-archive>

### 2.1 Windows 설치

1. 위 페이지에서 **Operating System: Windows → Architecture: x86_64 → Version: 11/10 → Installer Type: exe (local)** 선택.
2. 받은 `cuda_13.0.2_windows.exe` 실행 → "Express (권장)" 설치.
3. 설치 후 PowerShell에서 확인:

   ```powershell
   nvidia-smi
   nvcc --version
   ```

4. `nvcc`가 인식되지 않으면 시스템 환경 변수 `Path`에 다음을 추가:

   ```
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.0\bin
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.0\libnvvp
   ```

5. 재부팅한 뒤 다시 `nvcc --version`으로 확인.

> **드라이버 주의**: CUDA 13.x는 NVIDIA 드라이버 580 이상이 필요합니다. 구형 드라이버라면 GeForce Experience 또는 NVIDIA 다운로드 센터에서 먼저 업데이트하세요.

## 3. Python 환경과 저장소 준비

이 프로젝트는 [`uv`](https://github.com/astral-sh/uv)로 의존성을 관리합니다. Python 3.13을 사용합니다.

### 3.1 `uv` 설치

  ```powershell
  irm https://astral.sh/uv/install.ps1 | iex
  ```

설치 후 새 셸을 열어 `uv --version`이 출력되는지 확인하세요.

### 3.2 저장소 클론

```bash
git clone https://github.com/sorryhyun/anima_lora.git
cd anima_lora
```

> 이 가이드는 모든 경로를 `anima_lora/` 디렉터리 기준으로 설명합니다. 모든 명령은 이 폴더 안에서 실행하세요.

### 3.3 의존성 설치

```bash
winget install ezwinports.make
uv sync
```

`uv sync`는 `pyproject.toml`/`uv.lock`을 보고 가상환경을 만들고 모든 의존성을 설치합니다. 설치가 끝나면 다음 두 가지 중 한 방법으로 셸을 활성화합니다.

- 세션마다 활성화 (권장): `.venv\Scripts\activate`
- VSCode를 사용하면 비교적 편리하게 세션을 활성화할 수 있습니다.

## 4. Hugging Face 인증 및 모델 다운로드

### 4.1 토큰 발급 & 로그인

1. <https://huggingface.co/settings/tokens> 에서 **read** 권한의 토큰을 발급.
2. 터미널에서 로그인:

   ```bash
   hf auth login
   ```

   토큰을 붙여넣고 엔터.

### 4.2 모델 다운로드

```bash
make download-models
```

이 명령은 다음 세 가지를 `models/` 아래 자동으로 받아 정리합니다.

| 파일 | 경로 |
|---|---|
| Anima DiT (확산 모델 본체) | `models/diffusion_models/anima-preview3-base.safetensors` |
| Qwen3 0.6B 텍스트 인코더 | `models/text_encoders/qwen_3_06b_base.safetensors` |
| QwenImage VAE | `models/vae/qwen_image_vae.safetensors` |

마스킹 기능까지 사용할 예정이라면 SAM3와 MIT 모델도 함께 받아둡니다 (위 명령에 이미 포함됨).

> **다운로드가 끊긴다면**: `make download-anima`, `make download-sam3`, `make download-mit`처럼 개별 타겟으로 나눠서 다시 받을 수 있습니다.

## 5. 데이터셋 준비

Anima LoRA는 *이미지 + 같은 이름의 `.txt` 캡션 사이드카* 구조를 사용합니다. 아래는 `image_dataset/` 폴더의 예시입니다.

```
image_dataset/
├─ 00001.png
├─ 00001.txt
├─ 00002.jpg
├─ 00002.txt
├─ subfolder/
│  ├─ 00010.webp
│  └─ 00010.txt
└─ ...
```

### 5.1 캡션 작성 팁

- anima 공식 가이드라인을 따르면 태그 순서는 항상 [meta] [character] [series] [artist] [general] 입니다. 예를 들어

```
absurdres, safe, 1girl, chitanda eru, hyouka, @channel (caststation), full body, serafuku, She is saying hi.
```
와 같이 작성하시면 됩니다.

- 이 때 absurdres, highres, masterpiece 등의 퀄리티 태그는 개인적으로 실험한 결과 작성하지 않거나 최소한으로만 남기는 것이 제일 좋습니다. 혹은 추후 정식 배포될 mod guidance를 사용하면 아예 작성하지 않아도 됩니다.
- 원본 이미지는 `image_dataset/`에 둡니다 (이름은 자유, 이 위치를 그대로 사용하세요).

## 6. 전처리: 리사이즈 · 잠재변수 · 텍스트 임베딩 캐싱

학습 속도와 VRAM을 최적화하기 위해 **리사이즈 → VAE 잠재 캐싱 → 텍스트 임베딩 캐싱** 세 단계를 사전 수행합니다.

```bash
make preprocess              # 세 단계 모두 실행
# 또는 단계별로
make preprocess-resize       # 1) image_dataset/ → post_image_dataset/ 리사이즈
make preprocess-vae          # 2) VAE 잠재변수 .npz 캐싱
make preprocess-te           # 3) 텍스트 인코더 출력 .npz 캐싱
```

### 6.1 리사이즈가 하는 일

- VAE가 요구하는 픽셀 정렬(8의 배수)에 맞춰 이미지를 리사이즈
- (H/16) × (W/16) ≈ 4096 패치가 되는 *고정 토큰 버킷*에 자동 분류
- 손상된 이미지/캡션 누락 등을 검증하고 보고

### 6.2 잠재변수 캐싱

- 모든 이미지에 대해 VAE를 한 번만 돌려 결과를 디스크에 저장
- 학습 중에는 VAE를 GPU에 올리지 않으므로 VRAM이 크게 절약됨
- 스크립트: `scripts/cache_latents.py`

### 6.3 텍스트 임베딩 캐싱

- Qwen3 0.6B + LLM 어댑터 출력을 미리 계산
- `caption_shuffle_variants = 8`로 캡션 순서 셔플 변형까지 같이 캐싱 (학습 시 무작위 선택)
- 스크립트: `scripts/cache_text_embeddings.py`

> **캐시는 언제 다시 만들어야 하나요?**
> - 캡션을 수정했거나 이미지가 추가/삭제되었을 때
> - 토크나이저/패딩 관련 옵션을 바꿨을 때
> 캐시 폴더를 통째로 지우고 `make preprocess`를 다시 돌리면 됩니다.

## 7. GUI 사용법

PySide6 기반 GUI를 사용하면 설정 편집, 데이터셋 탐색, 학습 모니터링, GRAFT 큐레이션을 한 화면에서 할 수 있습니다.

```bash
make gui
```

GUI 주요 기능:

- **Config 편집기**: 프리셋(Win 8GB / Win 16GB 등)을 클릭 한 번으로 적용, 모든 키를 사이드 패널에서 직접 수정
- **데이터셋 브라우저**: 이미지/캡션을 미리보고, 잘못된 캡션을 즉석에서 고침
- **학습 모니터**: 손실/스텝 진행률 표시
- **GRAFT 큐레이션**: 후보 이미지를 격자로 보고 마우스로 좋고 나쁨 분류

GUI에서 시작한 학습도 결국 `train.py`를 호출하므로, 같은 파라미터를 CLI에서도 그대로 재현할 수 있습니다.

## 8. 학습 실행

모든 학습은 TOML 설정 파일과 HuggingFace Accelerate를 통해 실행됩니다.

### 8.1 빠른 시작

```bash
make lora            # 표준 LoRA, configs/training_config_plain.toml
make lora-fast       # 16GB GPU용 빠른 프리셋
make lora-low-vram   # 8GB GPU용
```

### 8.2. 마스크드 로스 (텍스트 버블 제외)

만화/만화풍 데이터에서 *말풍선이나 텍스트 영역*을 학습 손실에서 제외하면 결과가 훨씬 깨끗해집니다.

```bash
make mask          # SAM3 + MIT 둘 다 생성 후 병합 → masks/
make mask-sam      # SAM3만
make mask-mit      # MIT/ComicTextDetector만
make mask-clean    # 모든 마스크 폴더 삭제
```

결과 PNG는 흑백입니다: **흰색(255) = 학습 대상**, **검정(0) = 제외**.

* 마스크 파일이 없으면 자동으로 무시되는 구조라서, 굳이 사용하지 않으셔도 됩니다.

### 8.3 주요 설정값

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `network_dim` | 16~32 | LoRA 랭크. 클수록 표현력↑, 파라미터↑ |
| `learning_rate` | 5e-5 | 학습률. T-LoRA/DoRA는 더 낮춰도 됨 |
| `max_train_epochs` | 2~64 | 데이터셋이 작을수록 에폭↑ |


### 8.4 학습 진행 중 일어나는 일

1. 텍스트 인코더 로드 → 캐시 생성 → 언로드
2. VAE 로드 → 캐시 생성 → 언로드
3. DiT를 *지연 로딩*하여 캐싱 단계의 VRAM 충돌 방지
4. LoRA 네트워크를 DiT의 cross-attention/qkv/projection 모듈에 패치
5. 노이즈 샘플링 → DiT forward → flow-matching 손실 → backward → 수동 grad all-reduce → optimizer step
6. (선택) `validation_split`로 검증 손실 측정 및 샘플 이미지 생성

### 8.5 출력물

- 학습된 가중치: `output/anima_lora.safetensors` (또는 `output_name`에 지정한 이름)
- 체크포인트: `save_every_n_epochs` 주기마다 저장
- 검증 샘플: `output/sample/`

## 9. LoRA 변형 선택 가이드

| 변형 | 명령 | 언제 쓰나요? |
|---|---|---|
| **LoRA** | `make lora` | 처음 시도하는 사람, 안정적 |
| **LoRA-fast** | `make lora-fast` | 16GB VRAM, 빠르게 결과 보고 싶을 때 |
| **DoRA** | `make dora` | 같은 랭크에서 더 나은 학습 효율, 캐릭터 학습에 강함, 16GB 이상 사용 |
| **OrthoLoRA** | (`use_ortho = true`) | SVD 기반, 직교성 정규화로 안정적 학습 (Linear 전용) |
| **T-LoRA** | `make tlora` | 타임스텝별 랭크 마스킹, 디테일/스타일 균형 |
| **T-DoRA** | `make tdora` | DoRA + 타임스텝 마스킹 |
| **HydraLoRA** | `make hydralora` | MoE 다중 헤드 라우팅, 다양한 콘셉트를 한 LoRA에 |
| **Prefix Tuning** | `make prefix` | 12GB VRAM, ~1 step/s, LoRA 대안 |
| **Postfix Tuning** | `make postfix` | 어댑터 cross-attention에 학습 가능한 N개의 벡터 추가 |

각 변형의 자세한 옵션은 [`docs/training.md`](training.md), [`docs/hydra-lora.md`](hydra-lora.md), [`docs/prefix-tuning.md`](prefix-tuning.md)를 참고하세요.

> **호환성 메모**
> - DoRA와 OrthoLoRA는 함께 사용할 수 없습니다.
> - T-LoRA는 LoRA/DoRA 둘 다와 호환됩니다.
> - HydraLoRA는 `cache_llm_adapter_outputs = true`가 필요합니다.

## 10. 추론

### 10.1 가장 빠른 테스트

방금 학습한 LoRA로 즉시 샘플을 만들고 싶다면:

```bash
make test            # 가장 최근에 저장된 output/*.safetensors를 자동 사용
make test-spectrum   # Spectrum 가속(약 3.75x) 적용
```

### 10.2 일반 추론

```bash
python inference.py \
    --dit models/diffusion_models/anima-preview3-base.safetensors \
    --text_encoder models/text_encoders/qwen_3_06b_base.safetensors \
    --vae models/vae/qwen_image_vae.safetensors \
    --lora_weight output/anima_lora.safetensors \
    --lora_multiplier 1.0 \
    --prompt "masterpiece, best quality, an anime girl in a sunlit forest" \
    --negative_prompt "worst quality, low quality, blurry" \
    --image_size 1024 1024 \
    --infer_steps 30 \
    --guidance_scale 4.0 \
    --sampler er_sde \
    --flow_shift 1.0 \
    --seed 42 \
    --save_path test_output
```

자주 쓰는 플래그:

| 플래그 | 설명 |
|---|---|
| `--lora_weight` | 학습된 LoRA 경로. 여러 개 동시에 줄 수도 있음 |
| `--lora_multiplier` | LoRA 강도 (0.0~1.5) |
| `--image_size H W` | 출력 해상도 (예: `1024 1024`, `1024 1536`) |
| `--infer_steps` | 디노이징 스텝 수 (보통 20~50) |
| `--guidance_scale` | CFG 강도 (3.0~5.0 권장) |
| `--sampler` | `er_sde`, `euler`, `dpm++` 등 |
| `--seed` | 재현용 시드 |
| `--spectrum` | Spectrum 가속 활성화 |

추론 옵션의 전체 목록과 P-GRAFT 추론 방법은 [`docs/inference.md`](inference.md)에 있습니다.
