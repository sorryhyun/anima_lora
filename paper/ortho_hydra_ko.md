# Ortho-Hydra 한국어 해설

> 원논문: *Ortho-Hydra: Orthogonalized Experts for DiT LoRA — A structural fix for the MoE-LoRA cold-start deadlock* (Seunghyun Ji, 2026)
>
> 이 문서는 원논문(`ortho_hydra.tex` / `ortho_hydra.pdf`)을 비전문가도 따라올 수 있게 한국어로 풀어 쓴 해설본입니다. 수식은 핵심만 남기고, 직관과 "왜 이게 문제인지"에 집중했습니다.

## 한 줄 요약

여러 명의 화풍을 한 LoRA에 욱여넣으면 평균 화풍만 남는다. 전문가(expert)를 여러 명 두는 HydraLoRA로 풀려고 하면, **모든 전문가가 똑같이 영(0)에서 시작**하기 때문에 라우터(router)가 누구를 뽑아야 할지 평생 모른다. 이 데드락을 **수학적으로 분리된 출력 공간**(disjoint subspace)을 미리 깔아두는 방식으로 깬다.

---

## 1. 문제 상황

### 1.1 LoRA의 두 가지 약점

확산 트랜스포머(DiT)를 LoRA로 파인튜닝할 때, 학습 데이터에 화풍이 여러 개 섞여 있으면 두 가지 문제가 같이 터진다.

1. **방향 충돌**: LoRA가 만들어내는 잔차(`B·A·x`)가 원래 가중치가 잘 잡고 있던 방향(예: 해부학, 글자 렌더링)을 망치는 쪽으로 움직일 수 있다.
2. **평균화**: 단 하나의 `(A, B)` 쌍으로 K명의 화가를 동시에 표현하려 하면, 옵티마이저는 K명의 *공통 구조*를 잡는 부분공간을 찾는다. 결과는 누구의 그림도 아닌, 모두의 평균.

각각에 대한 기존 해법:

- **방향 충돌 → OFT 계열** (OFT, BOFT, PSOFT): LoRA의 잔차를 "구조적으로 직교"하게 제약해서 원본 가중치와 싸우지 못하게 한다.
- **평균화 → HydraLoRA**: 업프로젝션 `B`를 한 개가 아니라 E개(`B_1, ..., B_E`)로 두고, 라우터 `g(x)`가 입력에 따라 누구를 쓸지 정한다.

### 1.2 두 해법을 단순히 합치면 안 된다

OFT 스타일의 "공유된 직교 베이스" 위에 HydraLoRA를 얹으면, 새로운 문제가 생긴다. 이게 이 논문이 풀려는 **콜드스타트 데드락**이다.

## 2. 콜드스타트 데드락

### 2.1 무엇이 문제인가

HydraLoRA는 모든 전문가 `B_i`를 0으로 초기화한다 (LoRA 표준 관행). 그런데:

- 모든 `B_i = 0` → 모든 전문가의 출력이 똑같다 (0).
- 라우터는 "누가 더 잘하는지" 신호를 못 받는다 → 라우터는 균등 분포(`g_i ≈ 1/E`)에 머문다.
- 균등 라우터 하에서 모든 전문가가 **똑같은 그래디언트**를 받는다.
- 똑같은 그래디언트를 받으니 똑같은 방향으로 움직인다 → 영원히 같다.

**결론**: E개의 전문가를 두고 학습했지만, 결과적으로는 단일 LoRA를 E배 비싸게 학습한 것과 같다.

### 2.2 기존 미봉책의 한계

원래 HydraLoRA 논문은 두 가지로 대응했다.

1. **Expert warm-up**: 처음 몇 스텝 동안 매 스텝마다 무작위로 한 명의 전문가만 학습. 일정에 의존적이고, 라우터 자체에는 신호가 없다.
2. **σ-jitter (가우시안 노이즈)**: `B_i ~ N(0, σ²)`로 초기화해서 인위적으로 차이를 만든다. 노이즈는 손실 함수와 무관한 방향이라, 공유 베이스의 그래디언트가 곧 노이즈가 만든 비대칭을 평탄화시킨다.

실험 결과(Figure 1): σ=0.1로 강하게 흔들어도 1k 스텝 동안 라우터 엔트로피가 0.005도 안 움직인다. **사실상 효과 없음.**

## 3. Ortho-Hydra가 푸는 방식

### 3.1 핵심 아이디어 — 출력 공간을 미리 쪼갠다

이 논문의 한 줄짜리 트릭은 이거다:

> **모든 전문가가 같은 방향으로 글자를 쓰지 못하게, 처음부터 칠판을 E개 영역으로 나눠준다.**

구체적으로는:

1. 사전학습된 가중치 `W_0`의 SVD를 구한다 → `W_0 ≈ U Σ V^T`.
2. `U`의 상위 `E·r`개 좌특이벡터를 가져와 **E개의 직교 슬라이스**로 자른다.
   - 전문가 1: `U[:, 1:r]`
   - 전문가 2: `U[:, r+1:2r]`
   - ...
   - 전문가 E: `U[:, (E-1)r+1:Er]`
3. 각 슬라이스는 자기 영역 *내부에서만* Cayley 회전으로 학습된다.

이렇게 하면:

- 슬라이스끼리 직교 (`P_i^T P_j = 0` for `i ≠ j`)가 **학습 내내 정확히** 유지된다.
- 라우터의 전문가별 점수가 0스텝부터 자연스럽게 **다르게 나온다**. 왜냐하면 각 전문가가 출력 공간의 다른 영역에 글을 쓰기 때문에, 업스트림 그래디언트가 각 전문가의 라우터 가중치 `w_e`로 들어올 때 *직교 성분*이 살아남는다.

### 3.2 구성 요소

| 이름 | 모양 | 학습? |
|------|------|------|
| `Q_basis` (다운프로젝션) | `(r, d_in)` | 고정 (공유) |
| `P_bases` (전문가별 베이스) | `(E, d_out, r)` | **고정** (전문가별 분리) |
| `S_q` (공유 회전) | `(r, r)` | 학습 |
| `S_p` (전문가별 회전) | `(E, r, r)` | 학습 |
| `λ` (게이트 스케일) | `(1, r)` | 학습 |
| 라우터 가중치 | 약 `(r+d_σ) × E` | 학습 |

`S_q`와 `S_p`는 **반대칭 행렬**(skew-symmetric)로 학습되고, Cayley 변환

```
R = (I - A)(I + A)^(-1),    A = S - S^T
```

으로 직교 행렬이 만들어진다. Cayley는 모든 스텝에서 *정확히* 직교를 보장한다. 정규화 하이퍼파라미터 따위 필요 없다.

> 참고: 원논문은 PSOFT의 Neumann 근사 대신 `torch.linalg.solve`로 푸는 게 r ≤ 64에서는 비용 차이도 없고, Neumann 시리즈는 `||A|| > 1`이면 조용히 발산해서 직교성이 깨지는데 손실에는 안 보인다고 지적한다.

### 3.3 Forward 흐름

```
ℓ = x · Q_eff^T              # 다운프로젝션 (rank-r 병목)
g = softmax(라우터(rmspool(ℓ), φ(σ)))   # 게이트
ℓ ← ℓ ⊙ λ                    # 게이트 스케일
P_combined = Σ_e g_e · P_eff[e]
y = W_0·x + s · P_combined · ℓ
```

여기서 `Q_eff = R_q · Q_basis`, `P_eff[e] = P_bases[e] · R_p[e]`.

### 3.4 라우터를 왜 RMS pool로?

DiT의 시퀀스 길이 L은 약 4096. 이 길이에서 라우터에 어떤 표현을 넣을지가 의외로 결정적이다.

- **mean pool, raw input**: 평균은 zero-mean 신호를 `√L`로 죽인다. 풀링 후 표준편차 ≈ 0.008. 라우터에 사실상 상수가 들어간다 → 학습 안 됨.
- **max pool, raw input**: DiT 입력에는 DC 바이어스 채널이 80~96배 튀어나오는 outlier가 있다. bf16에서 softmax가 saturation → 학습 안 됨.
- **RMS pool, rank-r 병목 활성화**: 부호가 양수로 제곱되어 `√L` 캔슬이 안 일어난다. 그리고 rank-r 공간은 outlier가 없다 → **유일하게 학습이 되는 조합**.

세 조합 중 RMS+rank-r만 라우터 가중치 노름이 초기화 위치에서 움직였다. 나머지는 학습 끝까지 랜덤 초기화와 구별 불가.

## 4. 왜 직교 슬라이스가 데드락을 깨는가

수학적으로 라우터의 전문가별 점수(softmax 전)는

```
score_e = w_e^T · rmspool(x · Q_eff^T)
```

이고, `w_e`로 들어오는 그래디언트의 상대적 크기는 그램 행렬 `P_eff[i]^T · P_eff[j]`에 의해 결정된다 (1차 근사).

- **공유 베이스**: 모든 전문가가 같은 `P_basis`를 쓰면 `P_eff[i]^T · P_eff[j]`는 r×r 직교 행렬이고 0이 될 수 없다. 영-초기화에서는 항등행렬이다. → 모든 전문가가 같은 출력 방향, 그래디언트도 동일, 라우터 게이트는 1/E 근처, 전문가들도 똑같이 움직임. **데드락**.

- **분리된 슬라이스**: `P_bases[i]^T · P_bases[j] = 0` (`i ≠ j`)이면 `S_p`가 무엇이든 간에 `P_eff[i]^T · P_eff[j] = 0`. 전문가들이 서로 다른 출력 부분공간에 쓴다 → 업스트림 그래디언트의 전문가별 성분이 직교 → 라우터 가중치 `w_e`로 들어오는 사영이 *진짜로 다르다*. **0스텝부터 라우터에 신호가 있다.**

핵심은 그래디언트의 *크기*가 아니라 *방향*이 0스텝부터 비대칭하다는 것. 크기는 처음에는 `λ ≈ 0`이라 작지만, λ가 자라면서 자연스럽게 커진다.

### 언제 안 되는가

분리가 가능하려면 `min(d_out, d_in) ≥ E·r`이 필요하다. DiT의 어텐션/MLP 가중치(d ∈ {1024, 3072, 4096})에서 E=4, r ≤ 64 정도라면 항상 만족. 좁은 프로젝션에서는 공유 베이스로 폴백하는데, 이 경우 원래의 warm-up 일정에 의존해야 한다 (불안정).

## 5. 실험 결과

### 5.1 세팅

- **베이스 모델**: anima-preview3-base (28블록, 2048채널 DiT, 흐름매칭). 텍스트 인코더(Qwen3-0.6B)와 VAE는 동결.
- **데이터**: 약 2.4k장의 멀티-스타일 애니메 이미지. SAM3 + manga-text-detector로 만든 마스크로 텍스트 영역 제외.
- **옵티마이저**: AdamW, lr=2e-5, bf16, 흐름매칭 시그모이드 타임스텝 샘플링, 캡션 드롭아웃 0.1, T-LoRA 타임스텝 마스킹 사용.
- **HydraLoRA 위치**: 매 DiT 블록의 MLP 두 개 linear (총 56개 linear), E=12, α=32.
- **밸런스 손실**: Switch-Transformer 스타일, weight=5e-7, warmup ratio=0.4.

비교 대상 (모두 동일 데이터 / 옵티마이저 / 스케줄):

| 변형 | 베이스 | 초기화 | rank |
|------|--------|--------|-----|
| naive | 공유 | `B_i = 0` | 32 |
| jittered | 공유 | `B_i ~ N(0, 0.1²)` | 32 |
| **ortho** | 분리 슬라이스 | 영-초기화 + 분리 직교 | 48 |

### 5.2 결과 (Figure 1)

핵심 지표는 **평균 정규화 라우터 엔트로피** `H̄ ∈ [0, 1]`. 1이면 완전 균등(데드락), 0이면 one-hot.

- **naive**: 1k 스텝 내내 `H̄ ≈ 1`에 못 박혀 있음 (변동 < 0.005).
- **jittered**: σ=0.1로 흔들어도 거의 동일. naive 대비 미미한 추가 드리프트만.
- **ortho**: 약 200스텝 만에 균등 분포에서 이탈, 1k 스텝에서 `H̄ ≈ 0.81`, 28k 스텝에서 `H̄ ≈ 0.35`로 수렴. 12명의 전문가가 모두 살아있음.

훈련 손실은 세 변형 모두 ~0.08 수준으로 거의 동일. **차이는 라우팅에만 있다 → 옵티마이제이션 문제가 아니라 라우터 신호의 유무 문제.**

> rank 차이(48 vs 32)에 대한 caveat는 원논문이 명시: 콜드스타트 신호는 ~200스텝에서 포화되고, rank를 늘려도 공유 베이스 라우팅은 회복되지 않으므로 결론에 영향 없음.

## 6. 한계와 논의

### 6.1 출력 공간 제한

`ΔW`가 `W_0`의 상위 `E·r` 좌특이벡터 안에 갇힌다. NLP에서는 도메인 안에서 "튜닝"하는 거라 자연스럽지만, 새로운 캐릭터/화풍을 학습할 때는 주성분 밖의 방향이 필요할 수도 있다. **이 제한이 실제로 문제가 되는지는 본 논문이 답하지 않은 빈 칸**.

### 6.2 분리 슬라이스가 *유일한* 답은 아니다

분리 슬라이스는 "전문가끼리의 그램이 0"이라는 조건을 만족시키는 하나의 방법일 뿐. Hadamard나 구조적 랜덤도 만족시킬 수 있다. SVD를 고른 이유는 PSOFT와 같은 직관: 사전학습 가중치가 이미 capacity를 집중시킨 방향이 아마도 가장 쓸모있을 것이라는 *가설*. 결과로 입증되지는 않았다.

### 6.3 Soft specialization regularizer와의 비교

Guo et al. 2025의 "출력 직교 페널티 + 라우팅 분산 페널티"는 **이미 학습된** MoE LLM에서 라우터가 균등으로 끌려가는 걸 막는 용도다. 콜드스타트 시점에서는:

- 출력 직교 페널티는 모든 전문가의 출력이 0이라 그래디언트도 0 → 신호 없음.
- 라우팅 분산 페널티는 그래도 라우터 가중치로 그래디언트가 흐르긴 하지만, 자기 자신의 그래디언트가 전문가 인덱스에 대해 순열 대칭이라 대칭을 깰 수 없음.

**구조적 분리 슬라이스가 논리적으로 먼저고, soft 페널티는 그 위에 얹힐 때 의미가 있다.** 두 아이디어는 결합 가능하지만, 순서가 중요하다.

### 6.4 시그마 축 specialization은 안 됐다

라우터에 시그마(노이즈 레벨)를 4-bucket sinusoidal로 넣고, bucket별 밸런스 손실 계수를 주면 "초기 노이즈 단계 전문가 vs 후기 정제 전문가"가 자동으로 분리될 거라 기대했지만 **안 됐다**. 같은 이유다: bucket별 밸런스 손실도 균등 라우터에서는 전문가 인덱스에 대해 순열 대칭이라 specialization 축을 만들지 못한다.

해법은 똑같은 패턴: **구조적으로** 전문가를 시그마 밴드에 묶어버리는 것. 각 전문가 e를 밴드 `e mod B`에 할당하고, 밴드 밖 로짓을 `-∞`로 마스킹하면 끝. 분리 SVD 슬라이스와 결합하면, 같은 밴드 안의 전문가들은 출력 공간이 분리되고 다른 밴드의 전문가들은 시그마 커버리지가 분리된다. 코드에는 인터리브 레이아웃으로 들어가 있고, 밴드 경계는 디자인 레버 (균등 vs `[0, 0.5, 0.8, 1.0]` 같은 비균등). 본격 실험은 future work.

| **무엇** | HydraLoRA의 재매개변수화 — 공유 베이스를 SVD 기반 Cayley 회전 베이스로, 전문가 업프로젝션을 분리된 직교 슬라이스로 |
| **왜** | 콜드스타트에서 영-초기화 전문가들이 라우터를 균등 분포에 묶어버리는 데드락을 0스텝부터 깨기 위해 |
| **어떻게 검증** | 매칭된 데이터/옵티마이저/스케줄 하에서 라우터 엔트로피 다이내믹스 비교 (naive / jittered / ortho 3종) |
| **무엇이 미해결** | 멀티-스타일 fine-tuning에서의 *생성 품질*, E·r·밸런스 손실 weight 민감도, rank-매칭 비교 |

핵심 메시지: **콜드스타트 데드락은 학습 속도 문제가 아니라 공유 베이스의 그래디언트 기하의 성질이다. 노이즈로 흔들어서는 못 깨고, 출력 공간을 구조적으로 분리해야 깬다.**

## 부록 A — 용어집 (이 논문에 자주 나오는 표현)

- **Cold-start deadlock**: 처음에 모든 전문가가 같은 곳에서 출발해서 라우터가 누구를 뽑아야 할지 모르는 상태. 옵티마이저가 자력으로 못 빠져나옴.
- **Cayley 회전**: 반대칭 행렬 `A`로부터 `(I-A)(I+A)^{-1}`로 직교 행렬을 만드는 방법. 정규화 페널티 없이 정확하게 직교 보장.
- **Permutation symmetry (전문가 인덱스에 대한)**: E명의 전문가를 어떻게 번호를 다시 매겨도 모델 / 손실 / 그래디언트가 똑같다는 성질. 깨지 못하면 전문가들이 영원히 같다.
- **Switch-Transformer balance loss**: 모든 전문가가 비슷한 비율로 호출되도록 강제하는 손실. 전문가가 죽는(=한 명만 뽑히는) 걸 막지만, 콜드스타트 데드락은 못 푼다.
- **Disjoint subspace**: 부분공간들이 서로 영(0)에서만 만나는 상태. 여기서는 출력 공간 `R^{d_out}`을 E개 슬라이스로 자른 것.
- **Flow-matching**: DiT의 학습 목표. velocity field를 회귀하는 방식의 확산 모델 학습. 노이즈 예측(epsilon prediction)의 일반화.
